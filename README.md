# Credito Service

API RESTful com background service para integrar, consumir, persistir e consultar creditos constituidos.

## Tecnologias

- .NET 8
- ASP.NET Core Web API
- Entity Framework Core
- PostgreSQL
- Kafka
- Kafka UI
- Docker e Docker Compose
- xUnit, Moq e FluentAssertions

## Arquitetura

O projeto segue uma organizacao inspirada em Clean Architecture:

- `CreditoService.Domain`: entidades de dominio.
- `CreditoService.Application`: DTOs, validacoes, interfaces de servico, contratos de infraestrutura e casos de uso.
- `CreditoService.Infrastructure`: EF Core, PostgreSQL, Kafka e background service.
- `CreditoService.Api`: controllers, health checks, Swagger e pipeline HTTP.
- `CreditoService.Tests`: testes unitarios e testes de repository com EF InMemory.

Os DTOs seguem a convencao:

- `InputModel`: entrada de dados, como integracao de creditos.
- `ViewModel`: saida de dados, como consultas por NFS-e ou numero do credito.

Organizacao dos DTOs:

```text
src/CreditoService.Application/DTOs/
  Common/
    SuccessViewModel.cs
  CreditoConstituido/
    CreditoConstituidoInputModel.cs
    CreditoConstituidoViewModel.cs
```

## Como Executar Com Docker

Na raiz do projeto:

```bash
docker compose up --build
```

A API ficara disponivel em:

```text
http://localhost:8080
```

Swagger:

```text
http://localhost:8080/swagger
```

Kafka UI:

```text
http://localhost:8081
```

Health checks:

```text
http://localhost:8080/self
http://localhost:8080/ready
```

O Docker Compose sobe:

- API
- PostgreSQL com volume persistente
- Kafka
- Zookeeper
- Kafka UI

As migrations sao aplicadas automaticamente no container da API por meio da variavel:

```text
Database__ApplyMigrations=true
```

## Endpoints

### Integrar creditos constituidos

```http
POST /api/creditos/integrar-credito-constituido
```

Exemplo:

```bash
curl -X POST http://localhost:8080/api/creditos/integrar-credito-constituido \
  -H "Content-Type: application/json" \
  -d '[
    {
      "numeroCredito": "123456",
      "numeroNfse": "7891011",
      "dataConstituicao": "2024-02-25",
      "valorIssqn": 1500.75,
      "tipoCredito": "ISSQN",
      "simplesNacional": "Sim",
      "aliquota": 5.0,
      "valorFaturado": 30000.00,
      "valorDeducao": 5000.00,
      "baseCalculo": 25000.00
    }
  ]'
```

Resposta esperada:

```json
{
  "success": true
}
```

Status code:

```text
202 Accepted
```

O endpoint publica uma mensagem distinta no topico Kafka `integrar-credito-constituido-entry` para cada credito recebido.

### Consultar creditos por NFS-e

```http
GET /api/creditos/{numeroNfse}
```

Exemplo:

```bash
curl http://localhost:8080/api/creditos/7891011
```

### Consultar credito por numero do credito

```http
GET /api/creditos/credito/{numeroCredito}
```

Exemplo:

```bash
curl http://localhost:8080/api/creditos/credito/123456
```

## Background Service

O background service consome o topico Kafka a cada 500 milissegundos.

Para cada mensagem:

1. Desserializa o credito.
2. Valida o contrato de entrada.
3. Converte `simplesNacional` de `Sim`/`Não` para boolean.
4. Verifica duplicidade por `numeroCredito`.
5. Insere o registro individualmente no PostgreSQL.
6. Confirma o consumo da mensagem no Kafka.

## Banco De Dados

Tabela principal:

```sql
CREATE TABLE credito
(
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    numero_credito VARCHAR(50) NOT NULL,
    numero_nfse VARCHAR(50) NOT NULL,
    data_constituicao DATE NOT NULL,
    valor_issqn DECIMAL(15, 2) NOT NULL,
    tipo_credito VARCHAR(50) NOT NULL,
    simples_nacional BOOLEAN NOT NULL,
    aliquota DECIMAL(5, 2) NOT NULL,
    valor_faturado DECIMAL(15, 2) NOT NULL,
    valor_deducao DECIMAL(15, 2) NOT NULL,
    base_calculo DECIMAL(15, 2) NOT NULL
);
```

## Rodar Localmente Sem Docker

Suba PostgreSQL e Kafka localmente ou use apenas os servicos do Compose:

```bash
docker compose up postgres kafka zookeeper kafka-ui
```

Depois rode a API:

```bash
dotnet run --project src/CreditoService.Api/CreditoService.Api.csproj
```

Por padrao, `Database:ApplyMigrations` fica `false` fora do Docker. Para aplicar migrations manualmente:

```bash
dotnet ef database update \
  --project src/CreditoService.Infrastructure/CreditoService.Infrastructure.csproj \
  --startup-project src/CreditoService.Api/CreditoService.Api.csproj
```
